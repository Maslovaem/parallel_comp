# lab1
## pi_mpi
Параллельное вычисление числа π методом численного интегрирования. Разбивает интервал [0,1] на N частей, распределяет интервалы между процессами циклически. 

Компиляция и запуск: 
```
mpicc -o pi_mpi pi_mpi.c -lm && mpirun -n 4 ./pi_mpi 10000000
```
## latency_test
Измерение времени передачи сообщений между двумя MPI процессами. Выполняет ping-pong тест (первый процесс отправил сообщение, замерил время, второй принял и тут же отправил обратно, первый процесс снова зафиксировал время. Разница, делённая на 2 есть время коммуникации между процессами) с несколькими повторениями.

Компиляция и запуск: 
```
mpicc -o latency_test latency_test.c && mpirun -n 2 ./latency_test
```
## advection
### advection_seq.c
Последовательная программа решения уравнения переноса. Используется разностную схему "левый уголок".

Компиляция и запуск: 
```
gcc -o advection_seq advection_seq.c -lm && ./advection_seq
```
### advection_mpi.c
Параллельная программа для решения уравнения переноса. Задачи делятся между процессами, для обмена информацией на границах зон используется вспомогательный массив.

Компиляция и запуск: 
```
mpicc -o advection_mpi advection_mpi.c -lm && mpirun -n 4 ./advection_mpi
```
### plot_solution.py
Визуализация решения уравнения параллельной и последовательной программой.
Исполнение: 
```

```
### viz.py

## sort
### parallel_mergesort.c
Массив делится на несколько частей, сортировка каждого происходит на отделных процессах. Затем происходит слияние результатов.

Компиляция и запуск:
```
gcc -o parallel_mergesort parallel_mergesort_pthread.c -lpthread
./parallel_mergesort 10000000 4
```
### seq_qsort.c
Последовательная сортировка массива

Компиляция и запуск:
```
gcc -o seq_qsort seq_qsort.c -lm
./seq_qsort 10000000
```

Скрипт вызывает обе реализации алгоритмов (последовательную и параллельную) и отображает зависимость ускорения от числа нитей и размера массива.
Исполнение:

## integr
### integral.c
Программа для вычисления интеграла от sin(1/x) на промежутке от a до b. Для борьбы с осцилляциями вблизи нуля используется метод Симсона, если погрешность вычислений на 2 и 4 интервалах слишком велика, то интервал делится на 2, вычисления на нём делигируются новой нити.

Компиляция и запуск:
```
gcc -o adaptive_integral adaptive_integral_pthread.c -lm -lpthread
./adaptive_integral 0.1 1.0 0.000001 4
```
### compare.py
Скрипт, котрый сравнивает результаты интегрирования, полученные с помощью integr.c с тем, что выдают функции SciPy

Исполнение:
```
python3 compare.py
```
